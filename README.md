# Desafio: Configurando uma Instância de Banco de Dados na Azure

## Objetivo
Este laboratório tem como objetivo praticar o processo de configuração de uma instância de banco de dados na plataforma **Microsoft Azure**, consolidando os conceitos aprendidos e documentando a experiência.

## O que aprendi
Durante o desenvolvimento do lab, explorei os seguintes pontos:

- Como criar uma **instância gerenciada de banco de dados** no Azure.  
- Diferença entre opções de **SQL Database, MySQL, PostgreSQL e Cosmos DB**.  
- Configuração de **parâmetros básicos**: nome, região, versão do banco e recursos de hardware.  
- Definição de **usuário administrador e credenciais de acesso**.  
- Configuração de **rede e firewall** para permitir conexões externas de forma segura.  
- Importância de **alta disponibilidade e escalabilidade** em bancos de dados na nuvem.  
- Monitoramento de desempenho e custos com o **Azure Monitor**.  

## Passo a Passo Documentado
1. Acessar o **Portal do Azure**.  
2. Selecionar **Banco de Dados > Criar Instância**.  
3. Escolher o tipo de banco (ex.: **Azure SQL Database**).  
4. Definir **nome da instância, região e versão**.  
5. Configurar **usuário administrador e senha**.  
6. Ajustar **rede virtual e regras de firewall**.  
7. Revisar e criar a instância.  
8. Testar a conexão via **SQL Server Management Studio (SSMS)** ou outra ferramenta cliente.  

## Conclusão
Esse desafio me ajudou a compreender como configurar e gerenciar instâncias de banco de dados na Azure, reforçando conceitos de **elasticidade, alta disponibilidade, segurança e gerenciabilidade** em ambientes de nuvem pública.
